=== ISSUE 3 — Title font unreadable ===

The H1 page titles currently use Fraunces in a configuration that renders poorly. Likely causes:
- opsz axis set to 144 (display size) instead of 32-40 (text-title size)
- Italic variant at 32pt sacrificing legibility

Diagnose what's actually applied to .page h1 in full.css. Two fixes to try:

A. Keep Fraunces but explicitly set font-variation-settings: "opsz" 36 for H1 titles. This uses the text-sized optical variant which is more readable.

B. If A is not enough, switch H1 titles from Fraunces italic to Fraunces regular (non-italic) at 32pt. Keeps the serif character, gains legibility. Use italic ONLY for the page title decorative elements, not the H1 body.

Test both approaches. Pick the one where a reader can read titles like "What was overlooked today" without squinting.

The hero number on the cover (144pt) stays with opsz:144 — that's the only place where display opsz is appropriate.
