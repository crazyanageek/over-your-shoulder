=== 9. Fix Download PNG — "Tainted canvas" error ===

Confirmed error: "SecurityError: Failed to execute 'toBlob' on 'HTMLCanvasElement': Tainted canvases may not be exported."

Root cause: Google Fonts loaded from fonts.googleapis.com/fonts.gstatic.com taint the canvas when the SVG embeds them in a foreignObject, so toBlob() refuses to export.

Solution: BUNDLE THE FONTS LOCALLY. The user explicitly chose this option for a "whaou" PNG output with perfect typography.

Implementation:
1. Create /report/fonts/ directory
2. Download the four font files we actually use. Only WOFF2 (modern browsers including Chrome MV3). Use curl from https://fonts.googleapis.com/css2?family=... with a standard User-Agent header to get the WOFF2 URLs, then curl those files.
   Fonts needed:
   - Fraunces (regular 400, medium 500, italic 400, italic 500) → 4 WOFF2 files
   - Inter (400, 500, 600, 700) → 4 WOFF2 files
   - JetBrains Mono (400, 500) → 2 WOFF2 files
   Total ~400 KB bundled.
3. Write /report/fonts.css that @font-face declares each file from the local path.
4. Update /report/index.html: REPLACE the Google Fonts <link> tags with <link rel="stylesheet" href="fonts.css" />. No more fonts.googleapis.com or fonts.gstatic.com references.
5. Same for popup/popup.html.
6. Update manifest.json CSP: remove the fonts.googleapis.com and fonts.gstatic.com entries from style-src and font-src (they're no longer needed).
7. Rebuild the download function: it can now use the original foreignObject+canvas approach because the fonts are same-origin. No taint.

After implementation, test:
- Load the report → fonts render correctly on-screen (same as before)
- Click Download → PNG file lands in Downloads folder, fonts rendered correctly in the PNG
- Console shows no network request to fonts.googleapis.com / fonts.gstatic.com
- Open the downloaded PNG and confirm Fraunces/Inter are rendered properly (not fallback Georgia/system-ui)
