=== FIX A — FOOTER COMPACTION ===

In report/styles.css, find the .foot rules. Currently:

  .foot {
    padding: 0.75rem 1rem;
    display: flex;
    flex-direction: column;
    align-items: stretch;
    row-gap: 0.5rem;
    ...
  }

Change padding and row-gap:
  padding: 0.5rem 1rem;  (was 0.75rem)
  row-gap: 0.35rem;      (was 0.5rem)

In the button rules .share-btn.primary, .share-btn.secondary find their padding. Likely something like padding: 0.5rem 1rem. Reduce vertical padding:
  padding: 0.35rem 1rem (was 0.5rem)

This saves ~8 px on padding, ~3 px on row-gap, and ~6 px per button (2 buttons if they're side by side, or just the one visible). Total saving: ~24 px.

=== FIX B — DESTINATIONS-LIST SINGLE LINE ===

In report/styles.css, find .destinations-list. Currently uses flex-wrap: wrap which allows 2-line layout for 3+ countries.

Change:
  flex-wrap: wrap; → flex-wrap: nowrap;
  overflow-x: hidden;
  
Add to .destinations-list:
  width: 100%;
  
Add to .dest-item:
  flex-shrink: 0;  (prevent items from shrinking)
  
For safety if a user has 7+ countries (rare but possible), we need a graceful degradation. Add:
  .destinations-list {
    mask-image: linear-gradient(to right, black 85%, transparent 100%);
  }

This fades the right edge if content overflows, indicating there's more off-screen. With 3 countries like "UNITED STATES · 2,899" + "FRANCE · 453" + "CHINA · 162", total text width is ~420-450 px at 10.5px font size. Container is 568 px. So no overflow in current fake data, mask won't visibly apply.
