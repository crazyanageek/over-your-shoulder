Phase 04.7.3 — Fix destinations-list clipped by map-band overflow hidden

=== BUG DIAGNOSIS ===

The .destinations-list element (country breakdown below the map on the badge) is clipped when there are multiple countries. Root cause: .map-band has `overflow: hidden` and its grid row (1fr in .badge's grid-template-rows) doesn't grow to accommodate the destinations-list after the map SVG.

Measured evidence:
- .map-band: height 199.71px, clipped by overflow:hidden
- SVG occupies up to 175px (max-height on #worldmap)
- .destinations-list positioned at y=875 but .map-band ends at y=849
- List is ~26px below parent boundary, invisible

Effect: with 1 country, list fits barely. With 2-3+ countries, list is completely cut.

=== FIX ===

Three changes in report/styles.css.

Change 1: Reduce SVG map max-height to leave space for the country list:
  #worldmap {
    ...
    max-height: 175px;  →  max-height: 145px;
  }

Change 2: Remove overflow:hidden from .map-band to let natural flow:
  .map-band {
    padding: 0.9rem 1rem 0.8rem;
    border-bottom: 1px solid var(--ink);
    min-height: 0;
    overflow: hidden;  ← REMOVE THIS LINE
  }

Change 3: In the .badge grid, .map-band's 1fr needs a sensible min-height to prevent other rows from squeezing it when the user has many countries. Add explicit minimum:
  .map-band {
    ...
    min-height: 230px;
  }

(Removes the min-height: 0 which was working against the grid sizing, replaces with an absolute min that accommodates SVG + list even with 4-5 countries.)
