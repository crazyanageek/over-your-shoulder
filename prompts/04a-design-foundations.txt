=== base.css — reset and typography base ===

- Modern reset (box-sizing border-box, margin 0 on body etc.)
- Body: font-family var(--font-body), color var(--ink), background var(--bg), font-size var(--size-body), line-height var(--line-normal), font-feature-settings "ss01", "cv11" (OpenType refinements for Inter)
- Fraunces with optical sizing enabled (font-variation-settings "opsz" 144 for display sizes)
- Anti-aliasing: -webkit-font-smoothing: antialiased; -moz-osx-font-smoothing: grayscale
- Selection color: background var(--ink); color var(--bg)
- Headings: font-family var(--font-display), font-weight var(--weight-semibold), line-height var(--line-tight), letter-spacing -0.02em
- Paragraphs: max-width 60ch for readability
- Subtle entrance: body { animation: fadeIn var(--fade-in-duration) var(--ease) both; } @keyframes fadeIn { from { opacity: 0; } to { opacity: 1; } }

=== components.css — primitives ===

Build and style these 8 primitives, and demonstrate each one in index.html:

1. .hero-number — massive Fraunces numeral, size var(--size-hero), weight medium, line-height var(--line-tight), with the "/100" suffix in a much smaller inline size in ink-muted
2. .section-title — Fraunces display size, as per base headings
3. .eyebrow — tiny uppercase label in Inter, size var(--size-micro), letter-spacing 0.08em, ink-muted, above a section title
4. .lede — opening paragraph, Fraunces italic, size var(--size-lede), line-height var(--line-snug), color ink
5. .body-text — Inter, size var(--size-body)
6. .data-line — a row with label on the left, value on the right, monospace on the value (JetBrains Mono), thin rule between, padding var(--space-2) 0
7. .meter — a horizontal progress bar, height 2px, background rule, filled with ink. Variant .meter.alarm fills with alarm color. No rounded corners. Width comes from a --fill custom property (e.g., style="--fill: 67%")
8. .rule — a full-width 1px divider, color rule, margin var(--space-6) 0

=== index.html — the showcase page ===

Single centered column, content-max width, generous vertical rhythm. Display each primitive with its name as an eyebrow label above it, arranged top-to-bottom in the order listed. Use a mix of realistic content (Fake data: "Your Data Exposure Score" → 39, a sample punchline, a fake data-line "Requests routed to the United States" → 1,188, etc.) so we can feel how the primitives read with actual content.

Include at the very top of the page a small "Design System Preview — v0.1" eyebrow in muted ink.
