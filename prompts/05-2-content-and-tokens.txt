=== TOKENS TO WIRE NOW (CATEGORY 1 + 2) ===

Category 1 — Directly from existing storage or functions:

Basic counters (chrome.storage.local.oys_counters):
- {totalRequestsToday} → counters.total (currently counts all, not just today; leave as-is for now, rename semantic is fine)
- {totalRequests} → same
- {topCountry} → top key in counters.byCountry by value
- {topCountryShare} → rounded % of top country
- {crossBorderShare} → same as percentCrossingBorders
- {percentCrossingBorders} → count of requests whose destination country ≠ userCountry, divided by total
- {percentToChina} → count of requests to country "CN" / total, as %
- {topDestinationShare} → count of requests to country "US" / total, as % (for CLOUD Act section)
- {cityCount} → number of distinct cities in counters.byHost (resolve via HOST_GEO table in report.js)
- {countryCount} → number of distinct keys in counters.byCountry
- {foreignCountryCount} → countryCount minus (1 if userCountry is in byCountry, else 0)
- {userCountry} → from detectUserCountry() (country code)
- {countryName} → full name from COUNTRY_NAME map for userCountry

Vendor metrics:
- {vendorCount} → number of distinct keys in counters.byVendor
- {topVendorName} → top key in byVendor
- {topVendorShare} → rounded % of top vendor
- {secondVendorName} → second key in byVendor (null if only 1 vendor)
- {secondVendorShare} → rounded % of second vendor (0 if none)
- {topToSecondRatio} → topVendorShare / secondVendorShare, rounded to 1 decimal ("4.2x"). If only 1 vendor: "∞" or "N/A — single vendor"
- {vendorName} → topVendorName (used in mono-vendor legend)
- {vendorList} → prose list per editorial: 1 vendor = "ChatGPT"; 2 = "ChatGPT and Claude"; 3+ = "ChatGPT, Claude, and Perplexity" (Oxford comma)

Score metrics (from computeScore):
- {todayScore} → result.score
- {volumeScore} → breakdown.volume
- {categoriesScore} → breakdown.categories
- {geographyScore} → breakdown.geography
- {continuityScore} → breakdown.continuity
- {avgDailyRequests} → counters.total / daysObserved, rounded

Category-related:
- {categoriesCount} → number of distinct sensitive categories touched (look up how current score logic determines this)
- {categoriesList} → English grammar list of category names matching the count

Time metrics (from computeExposureStats):
- {minutesActive} → stats.totalMinutes
- {secondsPerRequest} → stats.secondsPerRequest

Observation period:
- {daysObserved} → countDaysActive(all) — same function already in use

Date tokens:
- {date} → current date formatted "19 April 2026"
- {time} → current time formatted "3:45 PM"
- {year} → current 4-digit year

Category 2 — Simple new calculations (implement in full.js):

Hourly distribution (requires iterating events by hour-of-day):
- {peakHour} → the hour (0-23) with the most events, formatted as "3 PM" or "9 AM"
- {peakHourRequests} → number of events in that hour
- {idleHours} → number of hours (0-23) with zero events
- {firstActivityTime} → time of first event today, formatted "8:47 AM"
- {lastActivityTime} → time of last event today, formatted "6:12 PM"
- {dayNightRatio} → ratio of events between 07:00–23:00 vs 23:00–07:00, formatted as "4.2x" or "all daytime" if zero nighttime

For the concentrated-activity variant (< 6h of active hours):
- {startHour} → first hour of the day with events
- {endHour} → last hour of the day with events

Volume percentile (approximation without historical data):
- {volumePercentile} → percentage of observed days whose request count was LOWER than today's. With 1 day observed, this is "100%" or "N/A". With 7 days, compute from stored per-day totals.

=== IMPLEMENTATION STRUCTURE ===

Recommended approach for full.js:

1. At page load, fetch chrome.storage.local.get(null)
2. Compute all tokens (Category 1 + 2) into a single tokens object
3. Walk the DOM and replace all text containing {tokenName} with the corresponding value
4. Walk the DOM and handle conditional blocks (each conditional block has data-variant-if="..." attributes that JS evaluates)
5. Update page-number elements if any dynamic
6. Set document.title with the current date

For the conditional variants in the HTML, a clean pattern is:

  <div data-variant-group="geography-section">
    <div data-variant-if="multiCountry">...variant A text...</div>
    <div data-variant-if="allDomestic">...variant B text...</div>
    <div data-variant-if="allForeign">...variant C text...</div>
  </div>

JS evaluates the condition once, removes the non-matching siblings, keeps the matching one.
